**cc65** is the dominant free cross-development suite for 6502-family systems — a C compiler, powerful macro assembler (ca65), linker (ld65), archiver, and simulator bundled together. It is the toolchain of choice for C-language MEGA65 development: [[MEGA65-mega65-libc]] is its primary runtime library, [[MEGA65-mega65-fdisk]] and [[MEGA65-mega65-freezemenu]] both build with it, and the [[wiebow.mega65.com]] cross-dev pipeline uses it extensively.

_All claims below are sourced from ../../raw/github/cc65-cc65.md unless otherwise noted._

## What it does

Provides a complete cross-development package for 65(C)02 targets: C compilation (`cc65`), assembly (`ca65`), linking (`ld65`), archiving (`ar65`), object conversion (`co65`), and disassembly (`da65`). Supports more than 20 platforms including C64, C128, NES, Atari, Apple II, Commander X16, and generic 6502 configurations. The MEGA65 is not a primary target but the suite is used by many MEGA65 C projects — the official MEGA65 `mega65` target config lives in mega65-libc's build system.

## Installation

Packages available via most Linux distribution package managers (`apt install cc65`), macOS Homebrew (`brew install cc65`), and Windows binaries. Build from source: `make` in the repo root. Documentation is SGML-based in `doc/`; the rendered HTML lives at https://cc65.github.io/doc.

## Key features

- **C compiler** — ANSI C subset targeting 6502; extensive use of zero-page for performance
- **ca65 macro assembler** — full-featured, supports 6502/65C02/65CE02 instruction sets; widely used without the C compiler
- **ld65 linker** — flexible linker configuration files (`.cfg`) per target; supports ROM/RAM segment mapping
- **Runtime libraries** — per-target C runtime and standard library in `libsrc/`; separates startup, I/O, and heap
- **Simulator** — co-running CPU simulation for testing without hardware
- **Samples** — per-target example programs in `samples/`

## Architecture

Monorepo: `src/` contains compiler, assembler, linker, and tools; `libsrc/` provides platform-specific C runtime for each target; `cfg/` holds linker configurations. The MEGA65 cross-dev chain typically uses ca65 (assembler) or the full cl65 driver. The `ca65` assembler accepts 65CE02 and 45GS02 opcodes when configured with `.cpu 45GS02`.

## Example usage

```sh
# Assemble a MEGA65 source file
ca65 --cpu 45GS02 -o main.o main.s

# Compile and link a C program using mega65-libc
make -f Makefile_cc65   # inside mega65-libc project
```

## Maintenance status

2648 stars, 498 forks, zlib license, active development (last push 2026-05-19), default branch `master`. Official releases infrequent (V2.19 from 2020); development HEAD is the recommended version for MEGA65 use. Mailing list and IRC (#cc65 on libera.chat) for support.

## Ecosystem

The foundation layer for C-language MEGA65 development. [[MEGA65-mega65-libc]] provides the MEGA65-specific runtime on top of cc65. [[Edilbert-BSA]] is an alternative assembler with native 45GS02 support and a simpler single-file design; it complements rather than replaces cc65 for assembly-only projects.
